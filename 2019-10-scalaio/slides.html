Typescript

* created by microsoft * powerful ☺ but unsound ☹ type system * structural types ☺☹ * fp is possible but hard and not idiomatic * most popular typed fronted language * bindings still dont work * unsafe - types lie - json ++++

ReasonML

* created by facebook * ocaml base & ecosystem * mutability? ++++

Elm

* Created by Evan Czaplicki * opinionated * redux model * separated from js * most popular typed fp * handicapped haskell (no typeclasses) * tooling * bus factor ++++

Purescript

* real haskell * hard time competing with Elm ++++

Fable

* F# -> JS ++++

Scala.js

* huge scala ecosystem * quite popular * bindings are not reliable * not approachable for js devs ++++ How about WebAssembly? ++++ How about WebAssembly? * not for general-purpose web * closer to scala-native, rust and other native languages ----

# Communication ++++

Communication:

 ## Good old rest
  • HTTP
  • Proper method
  • /user/{id}/cats
  • json
  • Very well known
  • Established standard
  • Not so efficient
  • Not so convenient for frontend
++++

Communication:

 ## Graphql * convenient for fronted * not so much for backend * nice code generation (everywhere) * a lot of tooling * can be exposed from db directly ++++

Communication: Graphql

 ## Example TODO ++++

Communication:

 ## RPC
Features:
  • easily shared between fronted and backend
  • convenient for the backend
  • problematic on the fronted
 Examples:
  • grpc-web
  • autowire
  • udash
++++

Communication:

 ## Shared REST
Features:
  • easily shared between fronted and backend if both are scala
  • requires the same stack on both sides
  • all the benefits and problems of REST
 Examples:
  • endpoints
  • tapir
  • typedapi
++++

Communication:

 ## Websockets & SSE ++++

Communication:

 ## Websockets & SSE
Websockets:
  • on top of raw TCP
  • distinct from HTTP
  • two directional
  • binary
 Server-sent events:
  • raw HTTP
  • part of HTML5
  • one direction
